This guide will walk you through setting up Rollipop in your React Native project.

<Callout>
  Rollipop is actively developed and tested with the latest versions of React Native, targeting the
  **Hermes JavaScript engine** for optimal performance and compatibility.
</Callout>

## For Agents

If you are an agent working with Rollipop, expand the guide below before changing a project.

<details>
  <summary>Agent quick reference</summary>

Rollipop documentation is available as plain Markdown for agent workflows. Append `.md` to any
documentation URL to read the Markdown version directly:

```txt
/docs/get-started/quick-start
/docs/get-started/quick-start.md
```

Use [`/llms.txt`](/llms.txt) as the documentation index. It lists official Rollipop docs as
Markdown links, so an agent can discover the relevant page before reading task-specific details.

For local project work, start with the bundled agent guidance:

```bash
rollipop agent
```

`rollipop agent` prints the recommended diagnostics workflow, including SSE endpoints, MCP setup,
external debugging integrations, and how to inspect build events.

Rollipop also ships bundled Markdown skills for common agent tasks:

```bash
rollipop skills
rollipop skills list
rollipop skills get <name>
```

Start with `rollipop skills get core`, then switch to a narrower skill such as `dev-server`,
`debugging`, `configuration`, `build`, `migration`, or `plugins` when the task calls for it.

When an agent needs structured diagnostics, run the dev server with MCP enabled:

```bash
rollipop start --mcp --reset-cache
```

The default endpoints are:

- SSE events: `http://localhost:8081/sse/events`
- SSE client logs: `http://localhost:8081/sse/client-logs`
- MCP server: `http://localhost:8081/mcp`

Adjust the URLs when using custom `--host`, `--port`, or `--https` options.

</details>

## Prerequisites

- Node.js 22.12.0 or higher

## Installation

You have two options for getting started with Rollipop: using the automated setup command or manual installation.

### Option 1: Automated Setup (Recommended)

The quickest way to get started is to use `@rollipop/init`:

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tab value="npm">```npx @rollipop/init```</Tab>
  <Tab value="pnpm">```pnpm dlx @rollipop/init```</Tab>
  <Tab value="yarn">```yarn dlx @rollipop/init```</Tab>
</Tabs>

This command will:

- Install Rollipop as a dev dependency
- Configure Android and iOS build scripts to use Rollipop CLI
- Create a basic `rollipop.config.ts` configuration file (if it doesn't exist)

### Option 2: Manual Installation

If you prefer to set up Rollipop manually:

#### 1. Install the package

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tab value="npm">```npm install --save-dev rollipop```</Tab>
  <Tab value="pnpm">```pnpm add -D rollipop```</Tab>
  <Tab value="yarn">```yarn add -D rollipop```</Tab>
</Tabs>

#### 2. Configure React Native CLI

Create or update the `react-native.config.js` file in your project root:

```js title="react-native.config.js"
module.exports = {
  commands: require('rollipop/commands'),
  /* Existing configurations */
};
```

This configuration makes Rollipop's commands available to React Native CLI, allowing you to use Rollipop seamlessly with your existing React Native build process.

<Callout type="info">
  If you don't have a `react-native.config.js` file in your project, create one in the root
  directory with the configuration above.
</Callout>
