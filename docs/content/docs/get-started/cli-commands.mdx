Rollipop CLI is partially compatible with the React Native CLI, allowing for seamless integration with existing React Native projects.

## start

Start the React Native development server.

```bash
rollipop start [options]
```

### Options

| Option             | Description                                            | Type      | Default     |
| ------------------ | ------------------------------------------------------ | --------- | ----------- |
| `--config`         | Path to the Rollipop configuration file                | `string`  | -           |
| `--host`           | Host to start the development server on                | `string`  | `localhost` |
| `--port`           | Port to start the development server on                | `number`  | `8081`      |
| `--reset-cache`    | Removes cached files                                   | `boolean` | -           |
| `--https`          | Enables https connections to the server                | `boolean` | -           |
| `--key`            | Path to custom SSL key                                 | `string`  | -           |
| `--cert`           | Path to custom SSL cert                                | `string`  | -           |
| `--no-interactive` | Disables interactive mode                              | `boolean` | -           |
| `--client-logs`    | Enable client log streaming (deprecated)               | `boolean` | `false`     |
| `--cache`          | If false, the bundler will not load or store any cache | `boolean` | -           |
| `--mcp`            | Enables the MCP server at `/mcp`                       | `boolean` | -           |

### Examples

```bash
# Start with default settings
rollipop start

# Start on a custom port with cache reset
rollipop start --port 3000 --reset-cache

# Start with MCP diagnostics enabled
rollipop start --mcp

# Start with HTTPS
rollipop start --https --key ./key.pem --cert ./cert.pem
```

## bundle

Build a JavaScript bundle for production or development.

```bash
rollipop bundle [options]
```

### Options

| Option               | Description                                   | Type      | Default |
| -------------------- | --------------------------------------------- | --------- | ------- |
| `--config`           | Path to the Rollipop configuration file       | `string`  | -       |
| `--entry-file` \*    | Path to the root JS file                      | `string`  | -       |
| `--bundle-output` \* | File path where to store the resulting bundle | `string`  | -       |
| `--platform`         | Target platform (`ios` \| `android`)          | `string`  | `ios`   |
| `--dev`              | Build in development mode                     | `boolean` | `true`  |
| `--minify`           | Override bundle minification                  | `boolean` | `false` |
| `--sourcemap-output` | File path where to store the sourcemap        | `string`  | -       |
| `--assets-dest`      | Directory path where to store assets          | `string`  | -       |
| `--cache`            | Enable or disable bundler caching             | `boolean` | `true`  |
| `--reset-cache`      | Removes cached files before building          | `boolean` | -       |

<small>* Required option</small>

### Examples

```bash
# Development bundle
rollipop bundle \
  --entry-file ./index.js \
  --bundle-output ./dist/main.bundle.js \
  --platform ios \
  --dev true

# Production bundle
rollipop bundle \
  --entry-file ./index.js \
  --bundle-output ./dist/main.bundle.js \
  --platform android \
  --dev false \
  --sourcemap-output ./dist/main.bundle.map \
  --assets-dest ./dist/assets
```

## agent

Print guidance for LLM agents working on Rollipop projects.

```bash
rollipop agent
```

The command prints a compact agent guide, including:

- bundled skill discovery commands
- default SSE and MCP endpoints
- MCP client setup and tool discovery guidance
- external debugging integrations
- the recommended build-event workflow for agents

It does not start the dev server or perform any actions by itself. Start the dev server with `--mcp` before connecting an MCP client.

Use `rollipop agent` as the first reference when wiring an agent into a local Rollipop dev server. For task-specific guidance, list the bundled skills and print the one that matches the current task.

## skills

List or print the Rollipop skills bundled with the package. Skills are Markdown task guides intended for coding agents.

```bash
rollipop skills
rollipop skills list
rollipop skills get <name>
```

`rollipop skills` and `rollipop skills list` print the available skill names. `rollipop skills get <name>` prints the Markdown content for a specific skill.

Start with `core` for general Rollipop usage.
